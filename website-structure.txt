1. PROJECT OVERVIEW
================================================================================

ZAAM Store is a full-stack e-commerce web application built with Next.js 16 (App
Router). It features a luxury-themed UI with gold/accent color scheme, 3D product
visualization using Three.js, and a complete shopping experience including product
browsing, cart, checkout, wishlist, user dashboard, and an admin panel.

The app uses MongoDB as its database (via Mongoose ODM) and Cloudinary for image
storage. Authentication is handled via JWT tokens with bcryptjs password hashing.

8. STATE MANAGEMENT
================================================================================

The app uses React Context API with localStorage persistence for 3 contexts:

AUTH CONTEXT (AuthContext.tsx)
-----------------------------
- State: user, loading, error
- Storage: JWT token in localStorage (key: zaam_token)
- Actions: login(), register(), logout(), updateProfile()
- On mount: checks for existing token, loads user via GET /auth/me
- Connects to NEXT_PUBLIC_API_URL via axios

CART CONTEXT (CartContext.tsx)
------------------------------
- State: items[] (id, productId, name, price, image, quantity, size, color)
- Storage: localStorage (key: zaam_cart)
- Derived: totalItems, subtotal, tax (8%), total (via useMemo)
- Actions: addItem(), removeItem(), updateQuantity(), clearCart()
- Handles duplicate detection (same product + size + color)

WISHLIST CONTEXT (WishlistContext.tsx)
--------------------------------------
- State: items[] (id, productId, name, price, image, addedAt)
- Storage: localStorage (key: zaam_wishlist)
- Derived: wishlistCount (via useMemo)
- Actions: addToWishlist(), removeFromWishlist(), isInWishlist(), clearWishlist()

NOTE: Zustand 5.0.14 is listed in package.json but not currently used in any
source files. It may be reserved for future migration or refactoring.

9. AUTHENTICATION & AUTHORIZATION
================================================================================

AUTHENTICATION
--------------
- Registration: POST /api/auth/register (creates user, returns JWT)
- Login: POST /api/auth/login (validates bcrypt hash, returns JWT)
- Token: JWT with {userId, role}, expires in 7 days
- Storage: localStorage (zaam_token)
- Password hashing: bcryptjs, salt rounds = 12

AUTHORIZATION ROLES
-------------------
- customer: Browse, cart, orders, wishlist
- admin: Admin panel, manage products/orders/coupons/users/categories

ENFORCEMENT
-----------
- Frontend: ProtectedRoute component checks user.role, redirects if unauthorized
- API: authenticateRequest() middleware verifies JWT from Authorization header
- Admin-only: adminOnly() middleware checks payload.role === 'admin'

================================================================================
10. DATA FLOW
================================================================================

BROWSING PRODUCTS
-----------------
Page load -> useEffect fetches /api/products (batch of 100) ->
data mapped to Product interface -> client-side filter/sort/pagination

PRODUCT DETAIL
--------------
URL param [id] -> useEffect fetches /api/products/[id] ->
also fetches related products by category

ADD TO CART
-----------
Click "Add to Cart" -> CartContext.addItem() -> updates localStorage ->
Navbar badge updates reactively via context

CHECKOUT
--------
Multi-step form (Shipping Info -> Review Order -> Payment Method) ->
Simulated order placement (2s delay) -> clearCart() -> Success screen
(NOTE: Currently simulated - does NOT call the API)

AUTHENTICATION
--------------
Login form -> AuthContext.login() -> axios POST to external/internal API ->
stores JWT in localStorage -> sets user in context -> redirect to home

IMAGE UPLOAD
------------
Drag & drop files -> ImageUploader POST to /api/upload via FormData ->
Cloudinary SDK streams to cloudinary -> returns {public_id, url, secure_url}

12. KEY FEATURES
================================================================================

1. 3D Product Visualization - Three.js product viewer and cards with orbit
   controls, auto-rotation, graceful WebGL fallback

2. Interactive Particle Background - Canvas-based particle system with mouse
   repulsion effect on the hero section

   reorderable gallery, primary image setting

6. Dark/Light Mode - Toggle with localStorage persistence

7. Responsive Design - Mobile drawer navigation, adaptive layouts

8. Category-Specific Fields - Dynamic form fields based on product category
   (sizes for shirts/pants, notes for perfumes, movement type for watches)

9. Admin Panel - Full CRUD for products/orders/coupons/users with analytics

10. Search & Filtering - Client-side: category, brand, price range, color,
    size, rating, search, sort, pagination. Server-side: search, category,
    sort, featured, new
